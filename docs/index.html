FEATURES

One command, full enumeration

35+ modules across 7 protocols. Multi-target CIDR scanning. Multi-credential comparison. Actionable next steps with auto-filled credentials.

35+ Enumeration Modules

Users, groups, shares, LAPS, Kerberoastable, delegation, ADCS, gMSA, GPP, PSO, SCCM, and more. Run all with -A or pick specific modules. Covers SMB, LDAP, MSSQL, RDP, FTP, NFS, and VNC.

ARCHITECTURE

Phased execution pipeline

Port prescan, SMB validation, auth probe, cache priming, then 36 independent modules via ThreadPoolExecutor. Proxy mode auto-adapts.

1

Port Prescan

100-thread TCP scan discovers live SMB hosts across CIDR ranges. Filters targets before enumeration begins. Supports IP ranges and target files.

2

Auth Validation

Probes null session, guest session, and provided credentials. Detects admin access. Multi-credential mode compares access levels across users.

3

Module Dispatch

36 independent modules via ThreadPoolExecutor with 15 workers. Cache priming with 7 parallel LDAP queries. Proxy mode reduces to 2 workers.

4

Reporting Engine

Catppuccin Mocha colored output. Actionable next steps with auto-filled credentials. Share access matrix in multi-credential mode. JSON and file export.

GETTING STARTED

Install and enumerate

Python package with zero external dependencies. Requires NetExec on PATH. Install via pipx or pip.

install.sh
# Install with pipx (recommended)
$ pipx install git+https://github.com/Real-Fruit-Snacks/Maelstrom.git

# Or standard pip
$ pip install git+https://github.com/Real-Fruit-Snacks/Maelstrom.git

# Run tests
$ pytest
usage.sh
# Anonymous enumeration
$ maelstrom 10.0.0.1

# Full enumeration with creds
$ maelstrom 10.0.0.1 -u admin -p pass -d CORP -A

# Pass-the-hash
$ maelstrom 10.0.0.1 -u admin -H aad3b:31d6c

# CIDR scan
$ maelstrom 10.0.0.0/24 -u admin -p pass
DETECTION AWARENESS

Know the boundaries

Hidden From

  • Target systems — read-only enumeration, no commands executed
  • Kerberos TGS — identifies SPNs via LDAP only, no ticket requests
  • Relay attacks — identifies opportunities only, no exploitation
  • Lateral movement — no implants, persistence, or beaconing
  • File residue — output files created with 0o600 permissions

Visible To

  • Network monitoring — SMB/LDAP/RPC connection metadata
  • Windows Event Log — authentication and enumeration events
  • EDR / XDR — NetExec behavioral signatures
  • IDS/IPS — high-volume enumeration traffic patterns
  • SIEM — correlated auth events across protocols
